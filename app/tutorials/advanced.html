Agent API Deep Dive

Architecture

The Agent API exposes all cycleCAD functionality as a JSON-RPC style interface. Any external system (AI agent, web app, CLI tool, or MCP server) communicates with cycleCAD via window.cycleCAD.execute().

Basic Usage

// All Agent API calls follow this pattern:
const result = await window.cycleCAD.execute({
    method: 'shape.cylinder',
    params: {
        radius: 25,
        height: 80,
        centered: true
    }
});

console.log(result); // { id: 'shape_0', success: true, ... }

10 Namespaces (55 Commands)

Namespace Commands Purpose
shape.* 12 commands Create geometry: cylinder, box, sphere, cone, torus, wedge, tube, plate, disk, ring, hexbolt, washer
feature.* 10 commands Edit geometry: extrude, revolve, fillet, chamfer, boolean, shell, pattern, mirror, sweep, loft
assembly.* 8 commands Manage components: add, remove, position, explode, constraint, mate, joint, query
render.* 6 commands Viewport control: snapshot, multiview, fitToSelection, setView, toggleGrid, toggleWireframe
validate.* 5 commands Design review: dfm (Design For Manufacturing), weight, cost, clearance, summary
export.* 4 commands Output formats: stl, obj, gltf, json
scene.* 4 commands Scene management: clear, undo, redo, getState
material.* 2 commands Material assignment: list, apply
annotation.* 2 commands Markup: pin, note
query.* 2 commands Data retrieval: edges, faces

Self-Describing Schema

Query the API to understand what commands are available and what parameters they accept:

// Get schema for all commands
const schema = await window.cycleCAD.getSchema();

// Returns:
{
    shape: {
        cylinder: {
            description: 'Create a cylinder',
            params: {
                radius: { type: 'number', required: true },
                height: { type: 'number', required: true },
                centered: { type: 'boolean', default: false }
            }
        },
        ...
    }
}

Example: Agent-Driven Design

// AI agent creates a motor mount bracket step-by-step
async function createMotorBracket() {
    const api = window.cycleCAD;

    // 1. Create base plate
    let base = await api.execute({
        method: 'shape.box',
        params: { width: 100, height: 80, depth: 10 }
    });

    // 2. Add mounting holes
    await api.execute({
        method: 'feature.pattern',
        params: {
            type: 'rectangular',
            rows: 2,
            cols: 2,
            spacing: 40
        }
    });

    // 3. Fillet edges
    await api.execute({
        method: 'feature.fillet',
        params: { radius: 2 }
    });

    // 4. Validate design
    const review = await api.execute({
        method: 'validate.dfm',
        params: {}
    });

    return review;
}
Advanced: The Agent API is transport-agnostic. In-browser it uses direct function calls. Over MCP it uses stdio JSON-RPC. Over REST it uses HTTP. The command semantics remain identical.

MCP Server Integration

What is MCP?

Model Context Protocol (MCP) is a standard for AI agents to discover and call tools. cycleCAD exposes all 55 Agent API commands as MCP tools, allowing Claude, other LLMs, and specialized agents to control CAD directly.

Starting the MCP Server

// From command line:
npx cyclecad-mcp --port 3000

// Or via Node.js:
const mcpServer = require('cyclecad/server/mcp-server');
await mcpServer.start({ port: 3000 });

console.log('MCP Server running on :3000');

Connecting an AI Agent

Claude and other AI systems can now discover cycleCAD tools via MCP:

// In your Claude/Agent context:
"You have access to the cycleCAD CAD engine via MCP tools."

cycleCAD tools available:
- shape_cylinder — Create a cylinder (radius, height, centered)
- shape_box — Create a box (width, height, depth)
- feature_fillet — Fillet edges (radius)
- validate_dfm — Design for Manufacturing analysis
- ... (55 tools total)

// Agent usage:
"Create a 50mm diameter, 80mm tall cylinder, then add a 5mm fillet"

// Claude calls:
1. shape_cylinder { radius: 25, height: 80 }
2. feature_fillet { radius: 5 }

Transport Options

Transport Use Case Example
stdio (default) Local agents, Claude, easy integration npx cyclecad-mcp
WebSocket Browser agents, real-time collaboration ws://localhost:3000
HTTP (request/response) Stateless cloud functions, Lambda POST /tool/{toolName}

REST API & WebSocket

Overview

The REST API provides HTTP endpoints for cloud-based CAD operations. All endpoints accept JSON and return JSON.

Core Endpoints

Endpoint Method Purpose
/api/v1/execute POST Execute any Agent API command
/api/v1/schema GET Get schema for all commands
/api/v1/models GET List saved models
/api/v1/models/{id} GET Get model state/geometry
/api/v1/models/{id}/export POST Export to STL/OBJ/GLTF
/api/v1/token/balance GET Get account token balance
/api/v1/token/spend POST Record token spend
/ws/live WebSocket Real-time events and collaboration

Example: Execute via REST

// POST /api/v1/execute
curl -X POST http://localhost:3000/api/v1/execute \
  -H 'Content-Type: application/json' \
  -d '{
    "method": "shape.cylinder",
    "params": {
      "radius": 25,
      "height": 80
    }
  }'

// Response:
{
  "success": true,
  "id": "shape_0",
  "bbox": { "min": [...], "max": [...] },
  "tokensUsed": 150
}

WebSocket Real-Time Events

// Connect to WebSocket
const ws = new WebSocket('ws://localhost:3000/ws/live');

ws.onmessage = (event) => {
    const msg = JSON.parse(event.data);

    switch (msg.type) {
        case 'operation_complete':
            console.log('Operation done:', msg.operation);
            break;
        case 'user_joined':
            console.log('Collab user:', msg.user);
            break;
    }
};

Rate Limiting & CORS

Configure rate limits and CORS in your deployment:

// config.json

$CYCLE Token Engine

Overview

$CYCLE is a dual-ledger token system where users spend tokens to create designs and earn tokens by sharing them. It's inspired by Claude's API pricing model but adds creator economics.

Three Tiers

Tier Cost Features
FREE €0/mo 50 tokens/month, basic AI, 10MB storage, no publish
PRO €49/mo 5,000 tokens/month, advanced AI, 1GB storage, publish models
ENTERPRISE €299/mo 50,000 tokens/month, priority support, unlimited storage, API access

Per-Operation Costs

Operation Tokens Notes
Simple shape (cylinder, box) 50
Fillet/Chamfer 75
Boolean operation 150
Assembly constraint 100
Design review (DFM) 200
Export to STEP 250
AI design suggestions 500 Per request

Discounts

  • Cache Discount: 10% for repeated operations within 24 hours
  • Batch Discount: 25-50% for bulk operations (10+ in a script)
  • Loyalty Discount: 5% per month as a PRO/ENTERPRISE user

Double-Entry Ledger

All token transactions are recorded in a cryptographically signed ledger:

{
    "transactions": [

            "type": "spend",
            "amount": 150,
            "operation": "feature.fillet",
            "note": "Motor bracket project",
            "hash": "0xabc..."
        },
        {
            "id": "txn_def456",

            "amount": 500,
            "source": "marketplace_sale",

Advanced: The ledger is immutable. Each transaction is cryptographically chained (hash of previous txn included in next). This ensures a tamper-proof audit trail.

Multi-Agent Workflows

Agent Swarms

Multiple AI agents can collaborate on a design, each with specialized skills:

const swarm = await window.cycleCAD.agent.createSwarm({
    name: 'Motor Bracket Design',
    agents: [
        {
            role: 'designer',
            model: 'claude-opus',
            // Creates geometry, refines shapes
        },
        {
            role: 'analyst',
            model: 'claude-sonnet',
            // Validates design, runs FEA simulation
        },
        {
            role: 'manufacturer',
            model: 'domain-model',
            // Optimizes for manufacturability
        }
    ]
});

// Swarm processes iteratively:
// 1. Designer creates initial geometry
// 2. Analyst reviews, suggests improvements
// 3. Manufacturer optimizes for CNC/FDM/etc
// 4. Repeat until converged

Caching

// Enable geometry cache (10% token discount for 24h repeats)
{
    "caching": {
        "enabled": true,
        "strategy": "lru",  // Least Recently Used
        "maxAge": 86400,    // 24 hours
        "maxSize": 500       // MB
    }
}

Render Optimization

  • Instancing: Repeated parts (bolt patterns) rendered as single instanced draw call
  • Frustum Culling: Off-screen geometry skipped
  • Occlusion Culling: Geometry hidden behind opaque parts not rendered
  • Normal Maps: Use for surface detail instead of high-poly geometry
  • Baking: Bake lighting for static geometry, only dynamic objects computed at runtime
Advanced: For assemblies >1000 parts, consider splitting into sub-assemblies and loading on-demand.

Building Plugins

Plugin Architecture

Extend cycleCAD with custom features using the Plugin API (FeatureScript-equivalent in pure JavaScript):

// plugins/custom-bracket.js
export const plugin = {
    name: 'Custom Bracket Generator',
    version: '1.0.0',
    author: 'Your Name',

    async install(cycleCAD) {
        // Register custom command in Agent API
        cycleCAD.registerCommand({
            method: 'custom.bracket',
            description: 'Create parametric bracket',
            params: {
                width: { type: 'number' },
                height: { type: 'number' },
                holes: { type: 'number', default: 4 }
            },
            handler: async (params) => {
                return await createBracket(params);
            }
        });
    },

    async createBracket(params) {
        const api = window.cycleCAD;

        // Create base plate
        let base = await api.execute({
            method: 'shape.box',
            params: { width: params.width, height: params.height, depth: 10 }
        });

        // Add holes pattern
        for (let i = 0; i < params.holes; i++) {
            await api.execute({
                method: 'feature.hole',
                params: { diameter: 6, depth: 10 }
            });
        }

        return { success: true, id: base.id };
    }
};

Registering a Plugin

// In app/index.html or via marketplace
await window.cycleCAD.plugins.install({
    url: 'https://github.com/user/cyclecad-bracket-plugin/raw/main/dist/bracket.js'
});

// Plugin is now available
await window.cycleCAD.execute({
    method: 'custom.bracket',
    params: { width: 150, height: 100, holes: 6 }
});

PR Guidelines

  1. Fork the repo and create a feature branch: git checkout -b feature/my-feature
  2. Make changes in a new module under app/js/ or enhance existing ones
  3. Add tests in app/test-agent.html for your feature
  4. Run the visual test agent and verify all tests pass
  5. Update documentation in relevant section of app/tutorials/advanced.html
  6. Commit with descriptive message: git commit -m "feature: add X capability"
  7. Push to your fork and open a Pull Request
  8. Maintainers will review within 48 hours

Code Style

  • Use ES6+ modern JavaScript (const/let, arrow functions, async/await)
  • Module-level scope: no global variables except window.cycleCAD
  • Comments for non-obvious logic; self-documenting code preferred
  • Follow existing naming conventions (camelCase for functions/vars, PascalCase for classes)
  • Include JSDoc comments for public APIs
