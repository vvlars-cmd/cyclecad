Architecture Overview

ExplodeView was originally built as a standalone 3D viewer for complex assemblies. By extracting its core rendering engine and integration layer, we've merged it into cycleCAD as a native mode—sharing the same Three.js scene, camera, and controls while maintaining complete feature parity.

Shared Scene Architecture

Instead of two separate applications, cycleCAD now runs both edit and viewer modes on the same Three.js canvas:

Key concept: One Three.js renderer powers both modes. When you toggle to Viewer Mode, the edit tools fade away and the ExplodeView tools activate—but the 3D camera, lights, and selection system remain synchronized.

// Initialize Viewer Mode sharing cycleCAD's viewport exports const viewerAPI = await initViewerMode({ scene: window.cycleCAD.viewport.scene, camera: window.cycleCAD.viewport.camera, renderer: window.cycleCAD.viewport.renderer, controls: window.cycleCAD.viewport.controls }); // Now both modes share the same 3D world const selectPart = (index) => viewerAPI.selectPart(index); const explodeModel = (amount) => viewerAPI.explodeParts(amount);

Mode Switching: Edit ↔ Viewer

cycleCAD operates in two complementary modes, each optimized for a different workflow. Switch between them with a single button click or keyboard shortcut (V).

✏️
Edit Mode
Design & parametric modeling. Create sketches, extrude, revolve, apply constraints, and build assemblies.
Sketch tools • Operations • Constraints • Parameters
👁️
Viewer Mode
Inspect & analyze. View assemblies, explode components, measure, annotate, and export documentation.
Explode • Section cut • Measurements • AI analysis

Seamless Transition

Both modes share the same 3D viewport and selection state. When you:

  • Switch to Viewer Mode → your model loads instantly with all 57 ExplodeView tools ready
  • Select a part in Viewer Mode → it remains selected when you switch back to Edit Mode
  • Rotate/zoom the view → both modes maintain the camera position
// Toggle between modes toggleViewerMode(); // Switches Edit ↔ Viewer // Check current mode if (isInViewerMode()) { console.log('Viewing assembly'); } else { console.log('Editing model'); } // Keyboard shortcut // Press 'V' to toggle

API Reference

All viewer functions accessible via window.cycleCAD.viewer or the API object returned by initViewerMode().

Model Loading

loadFile(url, type)

Load STEP, STL, OBJ, or GLB file into viewer.

url (string) — file path or URL
type (string) — 'step' | 'stl' | 'obj' | 'glb'
Returns: Promise<{parts, assemblies, bbox}>
getAssemblyTree()

Get hierarchical assembly tree with 6 levels of nesting.

Returns: Promise<Array<{partIdx, name, parentIdx, level, childCount}>>

Selection & Highlight

selectPart(partIdx)

Select a part by index, highlight with orange outline, open info card.

partIdx (number) — 0-based index
Returns: Promise<PartInfo>
getSelectedPart()

Get current selected part info: name, bbox, volume, weight, material.

Returns: Promise<{idx, name, bbox, volume, weight, material}>

Explode & Assembly

explodeParts(amount)

Set explode slider position (0 = collapsed, 1 = fully exploded).

amount (number) — 0-1
Returns: Promise<void>
getAssemblyState()

Get explode amount and visibility state of all parts.

Returns: Promise<{explodeAmount, visibleParts, hiddenParts}>

Clipping & Section

setSectionCut(axis, position, flip)

Enable/disable section cut on X/Y/Z axis.

axis (string) — 'X' | 'Y' | 'Z' | null (disable)
position (number) — 0-100 (percent of bounding box)
flip (boolean) — flip cutting direction
Returns: Promise<void>

Export & Analysis

exportBOM(format)

Export Bill of Materials.

format (string) — 'csv' | 'html' | 'json'
Returns: Promise<Blob> (auto-downloads)
exportPartAsSTL(partIdx, options)

Export single part as STL.

partIdx (number) — part index
options.scale (number) — scale factor (default 1)
options.binary (boolean) — STL format (default true)
Returns: Promise<Blob>
identifyPartWithAI(partIdx)

Use Gemini Vision API to identify part from screenshot.

partIdx (number) — part index
Returns: Promise<{partName, description, supplier, mcmasterLink}>

Annotations & Measurements

addAnnotationPin(position, text)

Add 3D annotation pin at world coordinates.

position ({x, y, z}) — world position
text (string) — annotation text
Returns: Promise<annotationIdx>
startMeasurement()

Enter measurement mode. Click 2 points for distance, 3 for angle.

Returns: Promise<void>

Visibility

hidePartByIndex(partIdx)

Hide part from view (with undo support).

partIdx (number) — part index
Returns: Promise<void>
showAllParts()

Make all hidden parts visible.

Returns: Promise<void>

Undo/Redo

undo()

Undo last action in viewer or edit mode.

Returns: Promise<void>
redo()

Redo last undone action.

Returns: Promise<void>

Mode Management

toggleViewerMode()

Switch between Edit Mode and Viewer Mode.

Returns: Promise<'edit' | 'viewer'>
isInViewerMode()

Check current mode.

Returns: boolean

Events

on(event, callback)

Listen for viewer events.

event (string) — 'partSelected' | 'modeChanged' | 'explodeChanged' | 'annotationAdded'
callback (function) — handler
Returns: void

Docker Infrastructure

cycleCAD with integrated ExplodeView is deployed as part of a 3-service Docker Compose stack.

Browser-based 3D viewer for standalone STEP files. Also available as npm module.

:8080

Parametric 3D modeler with integrated Viewer Mode. Full platform for design & analysis.

:3000

STEP Converter

FastAPI server converts STEP files to GLB format using CadQuery/OCP/cascadio backends.

:8787
