Philosophy

"Agent-First OS for Manufacturing." cycleCAD is built for AI agents first, humans second. Every feature has a JSON API. Users interface through text prompts, not mouse clicks. Agents can design, simulate, order, and manufacture parts autonomously.

4. Key Technical Decisions (ADRs)

Why Three.js?

Decision: Use Three.js r170 (ES modules via CDN) instead of Babylon.js or custom WebGL.

Rationale:

  • ✅ Zero build step — import ES modules directly from CDN (jsdelivr)
  • ✅ Mature ecosystem — 15+ years, millions of projects, extensive docs
  • ✅ PBR materials — realistic rendering with metalness/roughness
  • ✅ WebGL 2.0 — shadow maps, normal maps, deferred rendering
  • ✅ OrbitControls.js — production-ready camera navigation

Why OpenCascade.js?

Decision: Use OpenCascade.js WASM B-rep kernel (50MB download) instead of JSCAD or Manifold.

Rationale:

  • ✅ Real B-rep geometry — solids, not meshes. True booleans, fillets, proper topology
  • ✅ STEP/IGES support — parse and generate industry-standard CAD formats
  • ✅ Industrial-grade — used in Fusion 360, FreeCAD, AutoCAD
  • ✅ Lazy-loaded — 50MB WASM only downloaded when user imports STEP file

Why ES Modules via CDN?

Decision: Import all dependencies (Three.js, Plotly, QRCode, etc.) from CDN with import maps. No webpack, no build step, no node_modules.

Rationale:

  • ✅ Zero build complexity — edit app.js, refresh browser, changes are live
  • ✅ Browser-native — ES6 modules are a web standard (2015)
  • ✅ Smaller artifact — no webpack overhead, tree-shaking by browser
  • ✅ Easy versioning — pin Three.js to r170 in import map, instant updates

Why Gemini Flash?

Decision: Use Google Gemini Flash (multi-modal) instead of GPT-4 or Claude.

Rationale:

  • ✅ Vision API — understand sketches, screenshots, part images
  • ✅ Fast + cheap — $0.075/1M input tokens (vs GPT-4: $30/1M)
  • ✅ Multimodal — text + image in same prompt
  • ✅ 100K context window — entire design history in context

Why BroadcastChannel for Multiplayer?

Decision: Use BroadcastChannel API (same browser, same origin) for MVP multiplayer. Upgrade to WebRTC in Phase C.

Rationale:

  • ✅ Zero server — works peer-to-peer in same browser (dev + testing)
  • ✅ Instant MVP — test multiplayer UX without infrastructure
  • ✅ Upgrade path clear — swap BroadcastChannel for WebRTC+Firestore later

5. B-Rep Kernel Architecture

The B-rep (Boundary Representation) kernel is the heart of cycleCAD's geometry engine. It's powered by OpenCascade.js, a 50MB WASM package compiled from the OpenCascade C++ library.

WASM Loading Pipeline

User imports STEP
→
Lazy-load 50MB WASM
→
Parse STEP binary
→
Read B-rep topology
→
Tessellate to mesh
→
Render in Three.js

Geometry Operations

Operation Input Output Status
Extrude Face (2D sketch) Solid (3D box) Live
Revolve Face + axis Solid (rotated) Live
Fillet Edge + radius Solid (rounded) Live
Boolean Union 2 solids 1 solid (fused) Live
Boolean Cut Solid A − Solid B Solid A with hole Live
Shell Solid + thickness Hollow (thin-wall) Live
STEP Export B-rep shape .step file Q2 2026

Undo Stack

Every geometry operation is pushed to an undo stack. The stack stores:

{
  featureType: "extrude",
  parameters: { depth: 50, sketch: {...} },
  result: BRepShape,
  timestamp: Date.now()
}

Undo/redo works by replaying the feature list from the beginning (deterministic rebuilds).

6. AI Pipeline

cycleCAD has a 3-tier AI architecture: Cloud LLM (Gemini Flash), Local LLM (Groq Llama), and Offline NLP fallback. Users can use any tier; they trade cost/speed for accuracy.

Text-to-CAD Flow

User: "draw a box 50×30×20mm"
→
Detect intent: CREATE
→
Extract params: {w:50, h:30, d:20}
→
Call shape.box(50, 30, 20)
→
Create B-rep solid
→
Render in viewport

AI Tier Comparison

Tier Provider Cost Speed Accuracy Use Case
Cloud Google Gemini Flash $0.075/1M tokens ~2s 95% Complex, multi-step designs
Local Groq Llama 3.1 8B $0.02/1M tokens ~1s 85% Cost-conscious, latency-critical
Offline NLP + keyword matching Free <100ms 70% No internet, simple commands

7. Killer Features

1. Text-to-CAD with Real B-Rep Geometry

Type "design a cylindrical bearing bore 25mm diameter" and cycleCAD generates production-ready B-rep geometry. Unlike image generators, the output is editable, parametric, and manufacturable.

2. STEP Drag-and-Drop Import

Drag a .step file onto the viewport. cycleCAD instantly parses it (server-side STEP converter for >50MB files) and renders it. Exploded view, part selection, assembly tree all auto-generated.

3. GitHub Actions CAD Visual Diff

Commit a design to GitHub. Our GitHub Action renders both versions as 3D screenshots. Visual diff shows what changed — moved parts in orange, new parts in green, deleted parts in red.

# Render CAD diff on every commit
- uses: vvlars-cmd/cyclecad-diff@v1
  with:
    before: main
    after: HEAD

4. 35+ Parametric Parts Library

Built-in library of parametric parts: ISO bolts, DIN standard hardware, MISUMI components, bearing housings, motor mounts, etc. Drag-drop into assembly, adjust parameters, auto-BOM.

5. Real-Time Multiplayer (BroadcastChannel MVP)

Two tabs of cycleCAD in the same browser stay in sync via BroadcastChannel. Edit in one tab, see changes in the other instantly. Full upgrade to WebRTC+Firestore in Phase C.

6. Design for Manufacturing (DFM) Analysis

AI analyzes your design:

  • 🔴 Thin walls <1mm (risky for injection molding)
  • 🟡 Sharp internal corners (add fillet to avoid stress)
  • 🟢 Feasible for CNC milling with 3-axis

7. Inventory Tracking & Reorder

Design a part, get BOM, auto-search MISUMI/McMaster-Carr, place order via REST API. Tracks inventory in IndexedDB. Reorder when stock < threshold.

Multiplayer Sync (BroadcastChannel)

// Tab A (sender)
const channel = new BroadcastChannel('cyclecad-sync');
channel.postMessage({
  type: 'FEATURE_ADDED',
  payload: { featureType: 'extrude', depth: 50 }
});

// Tab B (receiver)
channel.onmessage = (e) => {
  if (e.data.type === 'FEATURE_ADDED') {
    operations.extrude(e.data.payload);
    viewport.render();
  }
};

10. Deployment Architecture

GitHub Pages (Free Tier)

cycleCAD.com runs on GitHub Pages, no server, no database:

  • Static HTML/CSS/JS served globally from GitHub CDN
  • All state in browser (IndexedDB + localStorage)
  • No backend = no downtime, no infrastructure costs

Docker (Self-Hosted / Enterprise)

# 3-service stack
version: '3'
services:
  cyclecad:       # nginx + app (port 3000)
  converter:      # FastAPI STEP→GLB (port 8787)
  cycledb:        # PostgreSQL for enterprise (port 5432)

npm Package

Embed cycleCAD in any web app:

npm install cyclecad
import cyclecad from 'cyclecad';

const app = new cyclecad.App({ container: '#viewport' });
app.shape.box(50, 30, 20);
app.export.stl('box.stl');

CLI Tool

npm install -g cyclecad
cyclecad design.cad --format stl --output box.stl
cyclecad --repl  # Interactive REPL

MCP Server (for Claude, Cursor, VS Code)

npx @cyclecad/mcp-server --port 3000
