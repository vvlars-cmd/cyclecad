System Architecture

cycleCAD is built on a LEGO microkernel architecture where the kernel provides a minimal set of core services and 21 specialized modules provide all user-facing functionality.

Data Flow Architecture

Event-Driven Flow

User Action (click, drag, keyboard) ↓ UI Layer captures event ↓ Emits: kernel.emit('action:type', {data}) ↓ Active Modules listen to event ↓ Command Pipeline: exec(command, params) ↓ Module executes: geometry update, state change ↓ Emits: kernel.emit('model:changed', {delta}) ↓ Subscribers update: - Viewport (re-render) - Tree (refresh) - Timeline (update keyframes) - Persistence (save) ↓ UI reflects changes

Command Execution Pipeline

Stage Description Example
Validation Check params, permissions, state Is extrude depth valid? Is user Pro tier?
Pre-hooks Any module can intercept Billing module charges user
Execution Module runs actual command geometry.extrude(sketch, depth)
State Update Update global + local state tree.addNode(), model.updateBBox()
Undo Push Save to undo history history.push({cmd, params, undo})
Post-hooks Modules react to change Render viewport, update BOM
Sync Broadcast if collaborative webrtc.broadcast({delta})

State Management

Three layers of state:

  • Global State: window.ccad = { model, assembly, config, auth, billing }
  • Module State: Each module can maintain local state (sketch.constraints, cam.toolpaths)
  • UI State: DOM attributes (data-selected, data-expanded)

Persistence Layer

Memory (scene graph) ↓ IndexedDB (local cache) ↓ Cloud Sync (if Pro) ↓ Version Control (collaborative) ↓ Backups (every hour)
