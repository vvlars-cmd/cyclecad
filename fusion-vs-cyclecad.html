How to Migrate from Fusion 360 to cycleCAD

1
Export Your Designs from Fusion 360
STEP Format (recommended):
File → Export → STEP (.step) — preserves geometry, loses parametrics

Native Format:
If you have Inventor files (.iam/.ipt), cycleCAD can import them directly (better than Fusion 360 can).

What you get: 3D geometry, assembly structure, part names, metadata
What you lose: Fusion 360 feature tree (rebuild operations), sketches, constraints
2
Import Into cycleCAD
Go to cyclecad.com/app
Click File → Import
Select your STEP file
cycleCAD will parse the geometry and create a 3D view

Note: Your design is now parametric again. You can add new operations, modify dimensions, export as STEP for CAM.
3
Verify Assembly Structure
Check the Model Tree (left panel) to see components
Use ExplodeView (integrated) to visualize assembly
Verify BOM (Bill of Materials) with AI auto-identification
Check assembly constraints — cycleCAD shows mate references if available

5
Known Limitations & Workarounds
Limitation 1: Fillet & Chamfer approximations
cycleCAD uses torus-based approximations, not real edge rounding (Fusion 360 uses actual BREP kernel).
Workaround: Export to STEP, use OpenCascade.js backend for real B-Rep boolean and fillet operations (in progress).

Limitation 2: Limited CAM (G-code)
cycleCAD has basic 3D slicing (FDM), not industrial CAM with 5-axis toolpaths.
Workaround: Export STL to Fusion 360 CAM, Fusion 360 CAM extension, or Kiri:Moto (free). cycleCAD integrates with Kiri:Moto one-click export.

Limitation 3: Simulation (FEA, Thermal)
cycleCAD doesn't have built-in FEA solver yet. Fusion 360 has mature simulation.
Workaround: Export STEP to free tools (FreeCAD, Salome, Code_Aster) or cloud FEA services (SimScale, OnScale).

Limitation 4: Sheet Metal unfolding
cycleCAD has sheet metal features (bend, flange, tab, slot) but unfold calculations are approximated.
Workaround: Use Fusion 360 CAM for flat patterns, or use specialized sheet metal software (Stratasys, Amada).

Fusion 360 → cycleCAD Win: Inventor Import
cycleCAD can import native Inventor .ipt/.iam files while Fusion 360 cannot. If you have legacy Inventor designs, cycleCAD is actually better for migration.
