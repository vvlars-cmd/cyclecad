Why we built it differently

Three things only an open browser-native platform can do.
🌐

Zero install

Open cyclecad.com on any device. No installer, no Inventor seat, no Windows-only restrictions. Field tech opens it on an iPad. Educator opens it on a Chromebook. Customer opens a share link in five seconds.

🔬

Deterministic engineering

Our AI Engineering Analyst computes every number with pure-JS analytical functions. The LLM only parses your prompt and writes the narrative. Math is auditable, deterministic, and verifiable against Shigley / VDI 2230 / AGMA / ISO 281 / AWS D1.1. 28/28 self-tests pass on every load.

🛠️

Real manufacturing

A design copilot that doesn't ship to a real machine is a dead end. Our CAM and Pentacad Bridge connect to a Pentamachine V2 over USB and stream live DRO via Rockhopper WebSocket. CAD → CAM → simulate → cut.

📂

Open formats, open code

STEP, GLB, G-code, NGC, DXF. All open. All standard. MIT / AGPL-3 source on GitHub. You can fork it, vendor it into your own product, or run it on your own intranet behind a firewall.

🤖

Agent-native API

window.cycleCAD.execute() exposes 55+ commands as a self-describing JSON-RPC interface. Claude, Gemini, or Groq can drive the entire Suite without screen scraping or pixel clicks. Macros are first-class.

🏗️

One pipeline, not three tools

Design in cycleCAD, present in ExplodeView, manufacture with Pentacad — all on one domain, one tab. Files cross between the three with a click. No imports, no exports, no version mismatches.

Want to see it run the same program MecAgent's site shows?

Open the milling demo — it runs the exact sim.pentamachine.com default G-code on the real V2-10 model, side-by-side with a narrative explaining each line. Or if you want the MecAgent-equivalent bolted-joint analysis, type analyze 4 M12 bolts class 10.9 with 18 kN shear and 18 kN axial in our AI Engineering Analyst — every number is derived from VDI 2230 / Shigley.

Honest about the gaps

Where MecAgent is ahead today.
  • Drawing automation — MecAgent's auto-dimensioning and orthographic projection generation is mature. Our drawing pipeline emits DXF and basic title-block layouts; auto-dimensioning is on the roadmap.
  • Inventor / SolidWorks integration — if you've already invested in Autodesk's stack and want AI inside it, MecAgent fits that workflow. Our story is "step out of Autodesk", not "live inside it".
  • Enterprise sales motion — MecAgent has a sales team, customer success, and probably an SSO story. We're a community + open-core project. If you need enterprise hand-holding today, that gap is real.
